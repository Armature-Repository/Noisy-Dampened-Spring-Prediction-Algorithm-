# Noisy-Dampened-Spring-Prediction-Algorithm

Overview

This project explores whether a neural network can infer the physical parameters of a damped harmonic oscillator from noisy position‑only measurements. The system follows the differential equation:
m\ddot {x}+b\dot {x}+kx=0
where:
- m = mass
- b = damping coefficient
- k = spring constant
- x_0 = initial position
- v_0 = initial velocity
The goal is to train a model that receives a noisy trajectory x(t) and predicts the underlying physical parameters (m,b,k,x_0,v_0). This is a classic system identification problem and is widely used in aerospace, robotics, and control engineering.

Motivation
Most machine learning projects rely on clean, perfectly labeled datasets. Real physical systems are not like that — sensors are noisy, dynamics are uncertain, and parameters must be inferred indirectly.
This project is designed to simulate that real‑world challenge:
- The model receives noisy, partial information
- The true parameters are hidden
- The mapping from trajectory → parameters is nonlinear
- The system evolves over time, requiring sequence modeling
This makes the project a strong demonstration of:
- physics‑informed machine learning
- time‑series modeling
- system identification
- LSTM‑based regression
- noisy data handling

Features
- Physics‑based data generator for damped harmonic oscillators
- Numerical integration of the ODE
- Additive Gaussian measurement noise
- Large dataset generation for supervised learning
- LSTM‑based neural network (SpringNet)
- Regression head predicting five continuous parameters

How It Works
1. Data Generation
Each sample is generated by:
- Randomly sampling m,b,k,x_0,v_0
- Numerically integrating the ODE using Euler’s method
- Adding Gaussian noise to the position measurements
- Storing the full trajectory x(t)
This produces a dataset of shape:
- X: (N, timesteps, 1)
- Y: (N, 5)
2. Model Architecture
The model is a simple but effective LSTM:
- Input: noisy trajectory
- LSTM layers extract temporal features
- Final hidden state → fully connected layer
- Output: predicted parameters
3. Training
The model is trained using:
- Mean squared error (MSE) loss
- Adam optimizer
- Mini‑batch training via DataLoader
A loss curve is saved as loss.png.
4. Prediction
Given a new noisy trajectory, the model outputs:
(\hat {m},\hat {b},\hat {k},\hat {x_0},\hat {v_0})

Example Output
Given a noisy trajectory, the model might predict:
Predicted: m=2.14, b=0.63, k=8.92, x0=1.21, v0=-0.44
True:      m=2.10, b=0.60, k=9.00, x0=1.20, v0=-0.50


Even with noise, the model learns to infer the underlying physics.
